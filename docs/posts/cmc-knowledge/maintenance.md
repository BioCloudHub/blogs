# CMC 知识库维护说明

> 这套知识库后续按“主题持续扩充”维护，不再按题号顺延管理。

## 维护目标

- 保留现有内容，不因为重排目录而丢失知识点。
- 取消页面中的题号式展示，避免后续新增内容时整库重编。
- 让新增、改写、补充都尽量落在现有主题结构里，而不是不断新开散页。

## 当前结构

- 一级专题页：用于按工作模块快速定位问题域。
- 二级主题页：用于收拢一类相近问题，减少重复维护。
- 细分主题页：承载具体知识点，适合连续阅读、搜索命中和逐步补充。

## 编辑规则

- 细分主题页中的知识点标题统一使用 `## 标题`，不再显示 `Q1`、`Q2` 这类题号。
- 每个知识点继续使用 `:::answer` 容器承载正文，方便搜索、阅读和统一样式。
- 新增内容优先补入最接近的现有细分主题页，不轻易新增平行目录。
- 若某一细分主题明显过长，再拆成新的细分主题页，而不是回头给全部知识点重新编号。

## 新增内容建议

1. 先判断内容属于哪个一级专题。
2. 再判断落在哪个二级主题最合适。
3. 进入对应细分主题页，直接插入新的 `## 标题` 和 `:::answer`。
4. 如果当前细分页已经过长，再考虑拆分主题，而不是恢复题号体系。

## 批量整理命令

```bash
pnpm cmc:organize
pnpm docs:build
```
